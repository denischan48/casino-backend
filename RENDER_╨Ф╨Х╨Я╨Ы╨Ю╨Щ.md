# Деплой бэкенда на Render — по шагам

Денис, это инструкция, как выложить бэкенд в интернет через Render.
Идём сверху вниз, ничего не пропуская.

---

## Шаг 1. Создать репозиторий на GitHub

1. Зайди на https://github.com (ты уже там залогинен).
2. Справа сверху нажми «+» → «New repository».
3. Repository name: `casino-backend` (можно любое имя без пробелов).
4. Оставь галку «Public» (или Private — без разницы).
5. Ничего больше не трогай, нажми «Create repository».

---

## Шаг 2. Залить файлы бэкенда в репозиторий (через браузер, без команд)

1. На странице нового репозитория найди ссылку «uploading an existing file»
   (синий текст в середине).
2. Распакуй архив `backend-deploy.zip`, который я тебе дал.
3. Открой распакованную папку. Выдели ВСЁ внутри неё (папку `app`, файлы
   `requirements.txt`, `.env.example`, `.gitignore` и остальные) и перетащи
   мышкой в окно браузера.
4. Дождись, пока все файлы подгрузятся (увидишь их список).
5. Внизу нажми зелёную кнопку «Commit changes».

> Важно: заливаешь СОДЕРЖИМОЕ папки, а не саму папку. В репозитории на верхнем
> уровне должны лежать `app/`, `requirements.txt` и т.д.

---

## Шаг 3. Создать сервис на Render

1. Зайди на https://render.com в свой аккаунт.
2. Нажми «New» → «Web Service».
3. Render покажет твои репозитории с GitHub. Выбери `casino-backend` → «Connect».

---

## Шаг 4. Настроить сервис

Render спросит настройки. Впиши так:

- **Name**: `casino-backend` (или любое)
- **Region**: оставь что предложит (ближе к Европе, если есть выбор — Frankfurt)
- **Branch**: `main`
- **Runtime / Language**: Python 3 (определит сам)
- **Build Command**:
  ```
  pip install -r requirements.txt
  ```
- **Start Command**:
  ```
  uvicorn app.main:app --host 0.0.0.0 --port $PORT
  ```
- **Instance Type**: выбери «Free»

---

## Шаг 5. Добавить переменные окружения

На той же странице пролистай до раздела «Environment Variables»
(или найди «Advanced»). Добавь по кнопке «Add Environment Variable»:

| Ключ (Key)    | Значение (Value)                     |
|---------------|--------------------------------------|
| `DEBUG`       | `true`                               |
| `BOT_TOKEN`   | (сюда вставь токен от @BotFather)     |

> Пока ставим `DEBUG=true` — чтобы проверить, что бэкенд вообще поднялся,
> без настоящего Telegram. Токен можно вписать сейчас или позже.

Нажми «Create Web Service». Render начнёт сборку (займёт 2-5 минут).

---

## Шаг 6. Проверить, что бэкенд живой

1. Когда сборка закончится, вверху появится адрес вида
   `https://casino-backend-xxxx.onrender.com`.
2. Открой этот адрес в браузере, добавив в конец `/docs`:
   `https://casino-backend-xxxx.onrender.com/docs`
3. Если видишь страницу с ручками API — БЭКЕНД РАБОТАЕТ. 

**Скопируй этот адрес и пришли мне** — он нужен, чтобы связать фронт с бэком
(следующий шаг).
